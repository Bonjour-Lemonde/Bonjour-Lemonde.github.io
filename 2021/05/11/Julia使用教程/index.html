Julia使用教程

1. 整体架构

1.1. 安装及VScode配置

  首先，笔者未进行软件的系统学习，对Julia的理解建立在试错基础上，专业术语为0.之后可通过提高专业技能完善该教程。
  在Julia上下载v1.4.2 (May 23, 2020) Windows 64(bit)（下载该版本的原因是，我需要使用的Package GpABC只在Julia1.4下进行过测试），允许默认的安装地址，将C:\Users\Dell\AppData\Local\Programs\Julia\Julia-1.4.2\bin添加至系统环境变量Path。
  在VScode中安装Julia，将C:\Users\Dell\AppData\Local\Programs\Julia\Julia-1.4.2\bin\julia.exe添加到Julia扩展设置里的Julia: Executable Path.至此Julia的搭建就完成了。
可能的Bug：
Building VectorizationBase → `C:\Users\XX\.julia\packages\VectorizationBase\LiMxH\deps\build.log` ┌ Error: Error building `VectorizationBase`: │ ERROR: LoadError: LoadError: InitError: could not load library "C:\Users\XX\AppData\Local\Programs\Julia\Julia-1.4.2\bin\LLVM.dll" │ The specified module could not be found.
Bug原因及解决办法：Julia路径中存在中文，换用英文的Julia路径。

1.2. Package的安装和使用

  Julia软件就像地基，我们需要运行的jl程序就像在地基上建房子，而已经封装好的包就像建房子用的各种材料。一般的安装方法如下，安装可以在任意文件夹进行。

  安装之后可以在当前文件夹中的Project.toml和Manifest.toml看到增加的包。使用该包的方法是运行using CSV，需要注意的是，第一次使用某个包，都需要先add，再using。之后再使用，比如关闭VScode之后再打开任意的文件夹并进入REPL模式，则直接运行using CSV即可。
  一般的卸载方法如下，有意思的是，卸载之后还能在C:\Users\Dell\.julia\packages看到CSV包，暗示包的彻底卸载也许需要手动删除？

  如果Pkg.add不成功，可以尝试develop，该方法无需进入要运行文件所在的文件夹，而是直接打开VScode并进入REPL模式，此时所在的文件夹应当是C:\Users\Dell\.julia。

  develop成功之后，包会存放在C:\Users\Dell\.julia\dev\GpABC，之后在VScode打开需要运行的jl文件所在的文件夹，无需添加包，运行using GpABC直接使用。之后发现包也会存在于C:\Users\Dell\.julia\packages。
  develop和add的区别：

  1. develop的包，里面的文件总是可读写的，而add的包，里面的文件可能是只读的，因此如果需要使用包里的example并需要进行修改的话，develop更好。
      包的更新使用Pkg.update("CSV")或者Pkg.update()，前者只更新CSV包，而后者更新所有包。如果更新所有包，可能会出现using GpABC失败的情况。
    可能的Bug：
    Pkg.update()前using GpABC可以正常运行，Pkg.update()后using GpABC运行失败。
    Bug原因及解决办法：更新之后GpABC依赖的包也更新了，如果GpABC与新版的包不兼容，则会出现using GpABC失败。解决办法是查看C:\Users\Dell\.julia\dev\GpABC\Project.toml里的Compat。 然后在这些包的Github pages上查看他们的最新版本，例如Colors的最新版本是0.12.7，可以兼容。而Optim的最新版本是1.3.1，不能兼容。解决办法是在Optim中加入1. 如何加参考教程。如果这种方式还不行，考虑删除C:\Users\Dell\.julia，并重新安装。目前还不会版本回退的方法。
